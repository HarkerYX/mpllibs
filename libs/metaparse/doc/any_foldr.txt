any_foldr
=========
Abel Sinkovics

Synopsis
--------

--------------------------------------
template <class P, class State, class BackwardOp>
struct any_foldr
{
  template <class S, class Pos>
  struct apply
  {
    // unspecified
  };
};
--------------------------------------


Description
-----------

Parser combinator taking a parser argument. It applies the parser on the input
string repeatedly as long as the parser accepts the input. The result of parsing
is equivalent to +boost::reverse_fold<Sequence, State, BackwardOp>+, where
+Sequence+ is the sequence of the results of the applications of +P+.

When the parser rejects the input for the first time, +any_foldr+ still accepts
the input and the result of parsing is +State+.

Header
------

--------------------------------------
#include <mpllibs/metaparse/any_foldr.hpp>
--------------------------------------

Expression semantics
--------------------

For any +p+ parser, +t+ class, +f+ metafunction class taking two arguments,
+s+ compile-time string and +pos+ source position

--------------------------------------
mpllibs::metaparse::get_result<
  boost::mpl::apply<any_foldr<p, t, f>, s, pos>
>::type
--------------------------------------

is equivalent to +t+ when +boost::mpl::apply<p, s, pos>+
returns an error. It is

--------------------------------------
boost::mpl::apply<
  f,
  mpllibs::metaparse::get_result<boost::mpl::apply<p, s, pos> >::type,
  boost::mpl::apply_wrap2<
    any_foldr<p, t, f>,
    mpllibs::metaparse::get_remaining<boost::mpl::apply<p, s, pos> >,
    mpllibs::metaparse::get_position<boost::mpl::apply<p, s, pos> >
  >::type
>
--------------------------------------

otherwise.

Example
-------

--------------------------------------
typedef boost::mpl::list<> empty_list;
typedef boost::mpl::push_front<boost::mpl::_2, boost::mpl::_1> push_front;

template <class P>
struct any : any_foldr<P, empty_list, push_front> {};
--------------------------------------
