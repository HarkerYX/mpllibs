accept
======
Abel Sinkovics

Synopsis
--------

--------------------------------------
template <class result, class remaining_string, class source_position>
struct accept
{
  // unspecified
};
--------------------------------------


Description
-----------

Compile-time data-structure describing the result of a successfull parsing. It
can be queried using link:get_result.html[get_result],
link:get_remaining.html[get_remaining] and link:get_position.html[get_position].

Its tag is +mpllibs::metaparse::accept_tag+.

Header
------

--------------------------------------
#include <mpllibs/metaparse/accept.h>
--------------------------------------

Expression semantics
--------------------

For any +a+, +b+ and +c+ classes, the following are equivalent

--------------------------------------
mpllibs::metaparse::get_result<accept<a, b, c> >::type
a::type
--------------------------------------

--------------------------------------
mpllibs::metaparse::get_remaining<accept<a, b, c> >::type
b::type
--------------------------------------

--------------------------------------
mpllibs::metaparse::get_position<accept<a, b, c> >::type
c::type
--------------------------------------

Example
-------

--------------------------------------
accept<boost::mpl::int_<13>, boost::mpl::string<>, mpllibs::metaparse::start>
--------------------------------------
